tester.js

Usage : node tester.js (all tests run automatically)

# Info Expected result
1 Tests myUse (upload of a file). A file called 'uploadMe' should be uploaded.
2 Checks parameters extraction (both by using ':' and by using '?'). Also checks that once a parameter appears, a regex is used for the resource. In addition cookies are tested. Code 200 should be returned.
3 Request resouce that was registered using 'post', where the request is of type 'get'. Error Code 404.
4 Checks only 'get' use - dynamicserver.get(). Code 200 should be returned.
5 Checks only 'post' use - dynamicserver.post(). Code 200 should be returned.
6 Checks only 'put' use - dynamicserver.put(). Code 200 should be returned.
7 Checks only 'delete' use - dynamicserver.delete(). Code 200 should be returned.
8 Although the file does not exist, we expect to invoke the suitable resource handler. In case that non of the registered resource handlers are invoked, 404 response will be sent. Expected return code 200 - the reason is that there is a match for the handler. The handler it self should verify that such file does not exists - which is the responsibility of the admin who uses this file.
9 Checks DOS attack (attempt to access unauthorized path). Request won't be answered.
10 (Part of load.js): Checks for 400 simultaneous get requests. 400 successful replies.
11 (Part of load.js): Checks for 400 simultaneous get requests. requests are send right after creation. 400 successful replies.
